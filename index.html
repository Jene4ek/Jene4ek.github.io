    Methods

    Bill Payments Invoice generation

    The above command returns JSON structured like this:

    API to create bill payment invoices.

    HTTP Request

    POST Base URL+/bill-payment/invoice/create

    Headers

    Type Params Values
    HEADER Content-Type string application/json
    HEADER Authorization string Basic Token. Will be given to you.

    Raw Body

    Object Params Values
    header txName string CreateBillInvoice
    reqData amount Number
    reqData address String (Not required for Qiwi Wallet Payment.
    Required for card Payment, the last 4 digits of Card.)
    reqData platform string Strictly: QIWI
    reqData currency String Transaction currency that is supported. Currently we are supporting RUB
    reqData ip String (Required. I.P. address should be of User who is completing the payment).
    reqData client String (QIWI WALLET NUMBER/CARD which MUST specify exchange user at the stage of creating an exchange request.)
    reqData recipient String (Identifier in the recipient system. If it is bank card then the card number,
    if it is BTC then BTC wallet address where bitcoin will be sent. If it is Yandex - then YA wallet number)
    reqData c_from String (Currency from - use Codes of electronic currencies from https://www.bestchange.ru/wiki/rates.html to specify source currency for exchange)
    reqData c_to String (Currency to - use Codes of electronic currencies from https://www.bestchange.ru/wiki/rates.html to specify destination currency for exchange)
    reqData date String (Date of transaction 2019-01-15T10:04:59)
    reqData txn String (Number of exchange requests/order in exchange system)
    reqData user_agent String (Client User Agent Id)
    reqData email String (Client Email)
    reqData cid String (id of user in exchange system)
    reqData extra Array (array of object containing extra information )
    reqData paySource String ( Not mandatory , paySource can be qw,card,mobile,sovest )
    qw -- payment source will be qiwi wallet
    card -- payment source will be card

    Merchants Wallets List

    The above command returns JSON structured like this:

    List of all wallets currently assigned to the Merchant.

    HTTP Request

    POST Base URL+ /merchant/get-wallet-list

    Headers

    Type Params Values
    HEADER Content-Type string application/json
    HEADER Authorization string Basic Token. Will be given to you.

    Raw Body

    Object Params Values
    header txName string fetchWallets
    reqData provider (optional) string:[QIWI,YANDEX]
    Default: will get both wallets
